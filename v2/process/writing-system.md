# 執筆システム v2 — 状態機械・グリッド・工程（R1・R5・R12）

> v1の章マネージャー方式（Phase A〜D）と3台帳を継承し、**「義務」と「通読」**を加えたもの。
> 核心思想: **禁止則だけでなく履行義務を持つ**（true-requirements §2-1）。義務はエピソードグリッドの列として実装し、機械チェック可能にする。

## 1. エピソードグリッド仕様（M3で各季 episode-list v2 として作成）

各話1行。**このグリッドが状態機械の実体**であり、台帳・章足場・執筆ブリーフはここから生成する。

| 列 | 内容 |
|----|------|
| Ep（仮番号）／タイトル案 | 通し番号は季の話数確定後に振り直し |
| モード | 静・観察／動・出来事／感情・別れ（**章内で静3連続禁止をこの列で検査**） |
| POV | Ako-chan／動物（誰か）／少年 等 |
| Day・季節 | chronology §4 のガード準拠 |
| 核イベント | 1話1つ。「起きたこと」を一文で |
| 状態デルタ | この話で変わる状態（知識・関係・体・場所）。**デルタゼロの話は原則作らない** |
| 登場・配役 | 出るキャラ＋ローテーション義務の消化マーク（R5: Dalu/Gabu等） |
| 🔒台詞 | この話で植える／回収する確定台詞（回収話には引用元Epを併記=双方向） |
| **呼称義務** | この視点で**使うべき名前**／禁止の旧記述子（R1） |
| **状態義務** | 見せるべき状態（例: Suke-sanの老い・Lucaの落ち着き・ロボットの成長目盛り） |
| 縦糸義務 | series-arc レジストリの該当行 |
| 沈黙免除 | 声フロア免除フラグ（季の10〜15%以内） |
| 語数帯 | 季の帯（S1 270-330／S2 350-450／S3 450-600） |

## 2. 台帳 v2（4冊）

| 台帳 | 内容 | 出自 |
|------|------|------|
| A. 知識状態・命名 | 誰が・いつ・何を知るか／固有名の確立点（**取得ビート表**込み） | v1継承＋R1拡張 |
| B. 配置・動線 | 定位置・ステージング・移動の妥当性（夜の犬・南極Pod等の細則含む） | v1継承 |
| C. 数・量 | 数値・数え語の登録簿（人数宣言の集約方式含む） | v1継承 |
| **D. 状態・義務** | 章ごとのキャラ状態スナップショット（年齢感・体・関係・技能）＋各話の義務欄の総覧 | **v2新設**（グリッドから生成） |

## 3. 工程（章サイクル＋季サイクル）

```
Phase 0  オリエンテーション: v2文書を README の順で読む（decisions → requirements → chronology → characters → series-arc → briefs → guide → 本ファイル）
Phase A  章足場: グリッド該当章の行を確認 → 台帳Dの章スナップショット作成 → 義務欄の転記 → docが決めていない判断点のみユーザー確認
Phase B  執筆プロンプト生成（別セッション用・季の声仕様と義務欄を必ず含める）
Phase C  バッチレビュー: 機械スキャン（§4）→ 精読（モード・4ビート・③・核達成・🔒逐語・台帳4冊照合・義務履行）→ 修正 → word_count実測更新
Phase D  章クローズ: リンク連鎖・全話レンジ内・コミット
Phase E  【新設】章完了ごとの連読レビュー: 直近10話を続けて読み、新情報率・発話率・レジスター変化・名前の出現を検査（「20話連読で初めて見える欠陥」の早期捕捉）
Phase F  【新設】季完了時の通読レビュー: 定量指標（§5）＋快感原理ゲート（§6）＋縦糸レジストリの全行消化確認
```

- パイロット方式継承: 新デバイス（新キャラの声・新レジスター）は章着手前に2〜3話書いてトーン確認。
- 執筆順序は S1→S2→S3（縦糸は前方に植えるため。ただしグリッドは3季ぶん先に完成させる=M3）。

## 4. 機械スキャン集 v2（Phase Cで実行）

v1継承分:
- 文法ティア（季別 will/would/could・現在完了/過去進行の可否・過去完了/仮定法/分詞構文）
- `robot`・Obia代名詞（he/his/she/her）・三連形容詞・畳語tic・フラット副詞・cannot
- 語数 = frontmatter一致＋**季別レンジ**（270-330／350-450／450-600 に更新）
- リンク連鎖・連番・story_day非減少・同題衝突（全季横断）

v2新設:
- **声フロア**: 本文の二重引用符ペア数=0 の話を検出 → グリッドの沈黙免除フラグと突合
- **呼称義務**: 名前取得後の話（グリッドで判定）に対し禁止記述子をgrep（例: 少年POVで `the small dog|the red-brown dog|one little tortoise` 等）。逆方向（確立前の名前先取り）はv1方式を継承
- **固有名ゼロ検出**: 固有名が1つも出ない話を列挙 → 意図（沈黙免除級の設計）か事故かをレビューで判定
- **レジスター連続**: グリッドのモード列で「静」3連続を検出
- **状態義務**: 章に割り当てた状態描写（例: 老い・成長目盛り）の未消化をPhase Eでチェックリスト照合

## 5. 定量回帰指標（Phase F・季ごとの目標値）

| 指標 | S1 | S2 | S3 |
|------|----|----|----|
| 発話箇所/話（平均） | ≥3 | ≥2.5 | **≥2**（v1実績0.45の再発防止） |
| セリフゼロの話 | ≤12% | ≤12% | ≤15%（すべて沈黙免除指定済みであること） |
| 固有名ゼロの話 | 0 | 0 | **0**（v1実績53話） |
| 現在時間のイベント密度 | 3〜5/章 | 3〜5/章 | **≥2/章**（v1実績1.6） |
| 12〜18語文の比率（S3のみ） | — | — | 15〜25%（B1の音楽） |
| 季節語彙の偏り | 章配分どおり | 同 | 冬語彙の全話比 ≤35% |

## 6. 快感原理ゲート（章設計時に記入・Phase Fで検収）

各章の設計時、次の6項目それぞれについて「この章ではどのビートが担うか」を一行で埋める。空欄が2つ以上ある章は設計に戻す。

1. **前進**（世界・状況の新情報）
2. **声**（動物の肉声・キャラの掛け合い）
3. **輪の内側**（名前が呼ばれる・家族の温度）
4. **謎の牽引**（次を読みたくさせる開いた問い）
5. **感情の振れ幅**（章内の対比: 軽↔重・動↔静）
6. **キャラの能動**（誰かが選ぶ・決める・動く）
