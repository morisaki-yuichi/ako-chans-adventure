# 真の要求分析 — 指摘の本質とv2要件（2026-07-11）

## 0. 一言でいうと

v1は「書きながら発見し、あとから台帳・監査で守る」**ボトムアップ運用**だった。今回指摘された問題は別々の不具合に見えて、ほぼすべて同じ根から生えている——

**物語の「状態」（誰が何を知っているか・誰が何歳か・誰に何ができるか・誰が今どう見えるか）が設計時に一元管理されておらず、台帳には「してはならない」（禁止則）しかなくて「しなければならない」（履行義務）が存在しなかった。**

- 「名前を知ったのに使われない」（問題1）＝解禁は設計済みなのに**使用義務**がないので旧記述子が残った。実測: 名前解禁（Ep219相当）以降もCh27/28/30/32の6ファイルに "small dog" が残存。
- 「Suke-sanだけ老いない」（問題4）＝老いは**Dango個人の属性**として書かれ、全キャラに時間を流す**状態表**がなかった。
- 「Ako-chanの貢献が高すぎる」（問題2）＝能力の**出所と水準**が設計項目に存在しなかった。
- 「Gabu/Daluが1ターンで終わる」（問題5）＝登場は禁止されないが**義務化もされない**ので、テーマ上必須のキャラだけが紙面を占めた。

したがってv2の中核要求はこう言い換えられる: **書く前に状態機械（年表×キャラ状態×知識/呼称×能力）を設計し、各エピソードに禁止と義務の両方を割り当て、それを機械チェック可能にする。**

## 2. メタ診断 — 言語化されていなかった本質

### 2-1. 禁止の台帳から、義務の台帳へ
v1の継続性システム（3台帳）は「先取りしない・矛盾しない」に強く、「やるべきことをやる」に無力だった。P1/P4/P5はすべて**不作為**のエラーで、既存のチェックリストには項目自体が存在しない。→ v2は各話ブリーフに**義務欄**（使うべき名前・見せるべき状態・出すべきキャラ）を持ち、レビューで義務の履行を検査する（R1/R4/R5）。

### 2-2. 「時間」はこのシリーズの主題であって、事務ではない
経過年数の調整（G2）は辻褄合わせに見えて、実はシリーズの核心に触れている。この物語は**時間の流れ方が違う存在たち**の話である——速く老いる犬（Dango・Suke-san）、ほとんど変わらないリクガメ（Dalu・Gabu）、変わってゆく子（Luca・少年・Ako-chan）、そして時間を一周するロボット（Obia）。ユーザーの指定した「不変のDalu/Gabu×変わるLuca」の対比は、この主題の明示化にほかならない。→ 年表（R4）は制約ではなく**設計の第一文書**として先頭に置く。

### 2-3. 教材原理と物語原理の二重拘束を、レベル設計で解く
「間延び／描写不足」（G3）の正体は、CEFRレベルだけが上がって**紙面の器が上がらなかった**こと。多読設計の常道どおり器をレベルに連動させれば、S3の「声・名前・脇役」を削る圧力そのものが消える（R7が R1/R5 の前提条件になっている点が重要）。

### 2-4. S3診断の「快感原理」を全季の設計ゲートに昇格する
v1のS3診断が特定した6項目（前進・声・輪の内側・謎の牽引・感情の振れ幅・キャラの能動）は、S3の反省を超えて**全シーズンの章設計チェックリスト**として使える。「20話連読して初めて知覚される問題」を捕まえる通読レビュー工程も常設する（R11/R12。→ `process/writing-system.md`）。

### 2-5. 現実の動物への忠実さは、このプロジェクトの倫理
v1でのユーザー判断は一貫して「実在の動物に嘘をつかない」方向だった（リクガメの休息姿勢の修正・コーギー寸法のリアリティ・成長速度の管理）。今回の外見変更（D7）・老いの並行描写（D11）も同じ線上にある。→ v2では**種ごとの行動・成長・寿命リファレンス**をキャラバイブルに常設し、執筆時参照を義務化する（R9）。

## 4. v1に存在せず、v2で新設する文書・工程

1. **マスター年表**（chronology.md）— v1では年齢が world.md の表に散在し、S3内の経過が未定義だった。
2. **義務台帳（台帳D）**— 各話の「しなければならない」リスト。エピソードグリッドの列として実装。
3. **能力・貢献モデル**— キャラの技能とその出所の台帳。
4. **季の声仕様**— 季ごと1枚の文体・視点・温度の定義（新旧混在の恒久対策）。
5. **シーズン通読レビュー**— 章完了ごとの10話連読＋季完了時の通読を正式工程化。
6. **伏線→回収レジストリ**— 植える話・回収する話の住所を持つ表（先行設計）。
